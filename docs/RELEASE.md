# Release Standards

We utilize Semantic Versioning (`MAJOR.MINOR.PATCH`) to release application versions:

## Release Tag Format
Tags should always be formatted as:
`v[MAJOR].[MINOR].[PATCH]` (e.g. `v1.0.0`)

## Codename Categories
We use Chess-themed codenames:
- `v1.0.0` - Codename: **Patzer**
- `v2.0.0` - Codename: **Gambit**
- `v3.0.0` - Codename: **Fianchetto**
- `v4.0.0` - Codename: **Zugzwang**

## Release Asset Format
Pre-packaged Windows binaries should follow:
`smart-chess-v[VERSION]-windows-x64.zip`
`checksums.txt`
