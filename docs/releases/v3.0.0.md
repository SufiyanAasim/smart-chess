# Smart Chess — v3.0.0

> *Codename: Fianchetto*

---

## 📖 Overview

Fianchetto is the third milestone of Smart Chess, shifting focus toward user personalization, audio variety, and immersive desktop aesthetics. This release introduces a threaded audio controller and customized player profile cards integrated directly next to the player timers, providing a tailored desktop experience.

Following the core features of Patzer and the analytical capabilities of Gambit, Fianchetto elevates the aesthetic and sensory experience of playing desktop chess.

---

## 🎯 Objectives

This milestone was developed with the following goals:

* Introduce dynamic, selectable audio feedback and sound controllers.
* Implement sleek player cards inside the game board sidebar.
* Enable user-customized player names across all game modes.
* Polish visual transitions and game-start layouts.

---

## ✨ Added

### 🎵 Custom Audio Controller & Sound Themes

An expanded sound engine framework allowing players to enjoy responsive acoustic feedback:
* **Thread-Safe Audio Engine**: Threaded Pygame mixer playback (`SoundManager`) ensuring zero UI latency or freezing during rapid move exchanges and captures.
* **Acoustic Cues**: Distinct auditory triggers for normal moves, piece captures, checkmate victories, and UI button interactions.
* **Volume & Toggle Controls**: Real-time master volume adjustments and sound enable/disable states persisted across user sessions.

---

### 💎 Custom Player Profiles & Naming

Sleek visual cards displaying player details next to the match timers:
* **Custom Player Names**: Registration across Local 1v1, Player vs Computer (`Stockfish 18`), and LAN multiplayer rooms.
* **Sidebar Profile Layouts**: Clean visual hierarchy placing player identities right above their respective digital clocks.

---

## 🔄 Changed

### ⚙️ Expanded Settings Interface

Redesigned sidebar configuration and options flow to support adjusting master audio toggles without interrupting live matches (`UISidebar`).

---

## 🏛️ Architecture Progress

Fianchetto introduces clean state management for sound assets and player identities:
* **Sound Controller Singleton**: Centralized audio routing mapping game board events (`move_piece`, `game_over`) directly to non-blocking playback threads.
* **Profile State Preservation**: Session storage preserving custom player names (`player_white`, `player_black`) across lobby transitions (`GameState`).
