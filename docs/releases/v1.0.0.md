Smart Chess — v1.0.0
Codename: Patzer

📖 Overview
Patzer represents the foundational milestone of Smart Chess, introducing a modern, secure, and highly responsive desktop chess workspace. This release establishes a solid foundation for offline AI play, local LAN multiplayer, and database-backed match tracking, packaged inside a custom flat-dark theme designed to optimize visibility and interaction.

By moving away from basic default Tkinter layouts and integrating advanced asynchronous operations, Smart Chess offers seamless gameplay controls, low-latency move hints, and isolated multiplayer room discovery. With a strong emphasis on architecture, data security, and standard packaging, this milestone lays the groundwork for modular feature development.

🎯 Objectives
This milestone was developed with the following goals:
- Establish a modernized, flat-dark graphical interface for chess play.
- Integrate Stockfish AI as a reliable, configurable local opponent.
- Implement real-time, asynchronous best-move recommendations.
- Enable serverless local network (LAN) multiplayer game rooms.
- Implement secure, SQL-injection-resistant local match logging.
- Structurally clean the codebase and organize code under the `src/` directory.

✨ Added
🎮 LAN Multiplayer Rooms
Introduced a decentralized local network multiplayer system, allowing players to host or join match lobbies over LAN without external servers.
Features include:
- Automatic LAN room discovery broadcasting over UDP (port 5000).
- TCP socket-based synchronization for game state and moves.
- Dedicated Spectator mode, allowing other local players to watch matches live.
- Integrated IP Address Helper providing clear connection tips on host configuration.

💡 Asynchronous Move Hints
Added real-time best-move calculations powered by local Stockfish analysis.
Features include:
- An asynchronous controller thread to prevent UI freezing during calculations.
- Neon-purple square highlights on the chessboard to display recommended paths.
- On-demand calculation triggered via a dedicated "💡 Get Move Hint" button.

🏆 Smart Match Ending States
Implemented a fully styled GameOver rematch overlay handling all termination rules:
- Traditional checkmate, stalemate, and draw states (threefold repetition, 50-move rule).
- Instant resignation and interactive draw agreements.
- Timeout detection synchronized with customizable match timers.
- Clean post-match popup to trigger instant rematches or return to setup.

💾 Parameterized Match Database & PGN Export
Introduced structured SQLite-backed history tracking to log matches locally inside `data/chess_games.db` (`11` historical matches consolidated) and export PGN files to `data/pgn/`.
Features include:
- Auto-recording of date, player names, outcomes, total moves, and game mode (`Local 1v1`, `LAN Host`, `Player vs Computer`).
- Fully parameterized queries to ensure absolute protection against SQL injections.
- Clean sidebar list integration showing past matches with PGN export capabilities.

🎨 Premium Modern Desktop Theme
Transformed the interface using professional design rules matching modern developer client aesthetics:
- Flat borderless sidebar elements and dark theme backgrounds.
- Active clock glows highlighted in neon-blue frames when a player is on turn.
- Low-time audio warnings and countdown animations.
- Smooth coordinates tooltips showing coordinate details on dock hover, complete with safe offsets to prevent flicker.
- Pawn promotion modal offering instant queen, rook, bishop, or knight selectors.

🔄 Changed
📦 Modern Modular Structure & Dual Path Handling
The application repository has been redesigned into a standard Python layout.
Major changes:
- Core execution modules moved to `src/`.
- Persistent read/write user data cleanly isolated under `data/` (`data/chess_games.db` and `data/pgn/`) and resolved using `get_data_path` so files persist alongside PyInstaller packaged executables (`SmartChess.exe`).
- Entry point consolidated in `main.py` which dynamically appends the search path.
- Config, specification assets, and documentation separated into dedicated resource directories (`src/resources/` using `resource_path`).

🔊 Audio Engine Threading
Migrated move sounds, captures, checks, and time alarms to a separate threaded mixer using Pygame. This avoids execution interruptions or stuttering during moves.

🚀 Improved
📁 File & Asset Resolution
Implemented robust relative path resolution in `src/utils_paths.py`, ensuring resources and binary icons load correctly regardless of from where or how the application is executed.

📊 Match Navigation controls
Enhanced scroll behavior in the match history logs with custom up/down navigation buttons matching the flat theme.

🏛️ Architecture Progress
Patzer prioritizes solid coding patterns and modern multi-threading setups to handle background processes.
Major structural progress:
- Modular architecture with clean splits between UI (`src/ui_board.py`, `src/ui_sidebar.py`), engines (`src/chess_engine.py`, `src/ai_controller.py`), networks (`src/lan_controller.py`), and data.
- Safe asynchronous loops for UDP broadcasting and Stockfish subprocessing.
- Complete separation of assets, binaries, test suites, and documentation.
