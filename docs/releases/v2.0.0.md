Smart Chess — v2.0.0
Codename: Gambit

📖 Overview
Gambit represents the next stage in the evolution of Smart Chess, extending the desktop client from a gameplay platform into an active learning workspace. This upcoming release introduces a post-match evaluation graph and real-time move classification markers, enabling users to analyze their tactical strengths and weaknesses directly within the application.

While the previous release (Patzer) focused on core engine integration and network capabilities, Gambit emphasizes local game analysis, move-by-move telemetry, and visual feedback systems to help players study past matches.

🎯 Objectives
This milestone is planned with the following goals:
- Introduce a visual post-match evaluation graph showing advantage swings over time.
- Implement move quality classifications (e.g., Best Move, Excellent, Book, Mistake, Blunder).
- Build interactive timeline navigation to review specific board positions.
- Improve post-game review performance using multi-threaded Stockfish analysis.

✨ Planned Additions
📊 Post-Match Evaluation Graph
An interactive chart displayed inside the game layout showing how the balance of power shifted during the match.
Features include:
- A centipawn-based advantage timeline tracking white and black superiority.
- Interactive nodes that update the board UI to the selected move when clicked.
- Visual thresholds highlighting critical momentum swings.

🏷️ Move Quality Classifications
A labeling system providing feedback on each played move.
Features include:
- Visual highlights on the board (e.g., red for blunders, orange for mistakes, green for excellent moves).
- Summarized game reports showing total counts of accuracy metrics.
- Detailed Tooltips explaining engine-evaluated alternatives for blundered moves.

🔄 Planned Changes
📈 Match History Enhancements
The local SQLite database schema will be updated to store centipawn evaluations and move classifications, enabling fast loading of past game charts without re-running Stockfish calculations.

🚀 Planned Improvements
⚙️ Background Engine Pooling
Optimize engine resource usage by pooling analysis processes, ensuring post-match analysis can run concurrently in the background without affecting UI responsiveness.

🏛️ Architecture Progress
Gambit continues to improve the application's clean separation of concerns by introducing an independent analysis engine module.
Major improvements planned:
- Dedicated evaluation manager decoupling engine calculation from UI controllers.
- Interactive event bindings between graph canvases and the primary board canvas.
- Extended database migration tools for handling evaluation persistence schemas.
